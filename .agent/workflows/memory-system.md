# 💾 记忆系统使用指南

## 三层记忆体系

本项目使用三层记忆系统，各司其职：

### 1️⃣ TODO.md（战略看板）
**用途**：项目任务管理
- 永久警告
- 已完成里程碑
- 当前任务清单
- 成功修复经验

**更新时机**：任务状态变化时

### 2️⃣ notes.md（施工日志）
**用途**：详细工作记录
- 调试日志
- 失败尝试
- 临时思考
- 交接文档

**更新时机**：每次工作过程中

### 3️⃣ memory.db（知识库）
**用途**：长期技术知识存储
- 技术技巧
- 解决方案
- 配置方法
- 常见问题

**更新时机**：发现值得记住的知识时

---

## 使用方法

### 存储知识到 memory.db

```bash
# 存储技术知识
python m.py add "你要记住的内容"

# 示例
python m.py add "Python虚拟环境：使用venv创建隔离环境，避免依赖冲突"
```

### 搜索知识

```bash
# 搜索关键词
python m.py search "关键词"

# 示例
python m.py search "Python"
python m.py search "Docker"
```

## 什么时候用哪个？

### 用 TODO.md 记录：
- ✅ 任务完成状态
- ✅ 项目里程碑
- ✅ 永久警告事项

### 用 notes.md 记录：
- ✅ 详细调试过程
- ✅ 报错信息
- ✅ 临时分析思路
- ✅ 交接文档

### 用 m.py 存储：
- ✅ 通用技术知识
- ✅ 可复用的解决方案
- ✅ 配置方法
- ✅ 最佳实践

## 工作流程示例

### 场景1：解决了一个技术问题

1. 在 `notes.md` 记录详细过程
2. 在 `TODO.md` 打勾完成
3. 用 `m.py add` 存储解决方案到知识库

```bash
# 示例
python m.py add "Nginx反向代理配置：proxy_pass http://localhost:3000; 解决跨域问题"
```

### 场景2：遇到新问题

1. 先用 `m.py search` 搜索知识库
2. 如果没找到，在 `notes.md` 记录探索过程
3. 解决后存入知识库

```bash
# 先搜索
python m.py search "Nginx"

# 解决后存储
python m.py add "新的解决方案"
```

### 场景3：新项目开始

1. 复制整个配置到新项目
2. 清空 TODO.md 和 notes.md
3. memory.db 保留（知识库可跨项目使用）

---

## 最佳实践

### 📝 记录原则

1. **TODO.md**：简洁明了，只记结果
2. **notes.md**：详细完整，记录过程
3. **memory.db**：通用可复用，记录知识

### 🔄 定期维护

- 每周清理 notes.md 过时内容
- 每月整理 TODO.md 已完成项
- memory.db 不需要清理（越积累越有价值）

### 💡 提示

- 存储到 memory.db 的内容要包含关键词
- 搜索时使用核心技术词汇
- 可以存储多个关键词的内容，方便检索
